Javascript promise, async, 그리고 await 공부1

Javascript promise, async, and await study

State

  • Pending -> Fulfilled or Rejected
  • Pending: 이행하지도, 거부하지도 않은 상태
  • Fulfilled: 연산이 성공적으로 완료됨
  • Rejected: 연산이 실패함

Producer and Consumers

  • Promise를 생성할 때, 2개의 callback을 받는데, Resolve와 Reject이다.
  • Promise는 생성되는 순간에 executer라는 callback 함수가 실행된다.
  • Promise는 Resolve나 Reject를 호출하지 안으면, 언제까지나 Pending 상태로 남아있다.
  • Promise는 Promise를 리턴하기 때문에 then, catch, finally 등을 사용하여 값을 출력한다.

Chaining

  • Then은 Promise를 반환하기 때문에 다시 catch를 호출할 수 있다.
  • 이때, then은 값(data)를 전달할 수 있으며, promise를 전달할 수도 있다.
  • 추가적으로 finally는 성공과 실패 여부와 상관없이 실행된다.

코딩테스트에서 자주 사용하는 Math methods (참고자료)

  • https://developer.mozilla.org/ko/docs/Web/JavaScript/Reference/Global_Objects/Math
  • 여러 메소드의 기초 사용법을 유형별로 정리한다.

Properties

Math.E

  • 자연로그의 밑 값 e를 나타내며, 약 2.718이다.

Math.LN10

  • 10의 자연로그 값, 약 2.302의 값이다.

Math.LN2

  • 2의 자연로그 값, 약 0.693의 값이다.

Math.LOG10E

  • e의 로그 10 값, 약 0.434의 값이다.

Math.LOG2E

  • e의 로그 2 값, 약 1.442의 값이다.

Math.PI

  • 원의 둘레와 지름의 비율, 약 3.14159의 값이다.

Math.SQRT1_2

  • 약 0.707의 값을 나타내는 루트 1/2이다.

Math.SQRT2

  • 2의 제곱근을 나타내고 약 1.414이다.

Methods

max

  • 입력값으로 받은 0개 이상의 숫자 중 가장 큰 숫자를 반환한다.

min

  • 입력값으로 받은 0개 이상의 숫자 중 가장 작은 숫자를 반환한다.

abs

  • 주어진 숫자의 절댓값을 반환한다.

pow

  • base에 exponent를 제곱한 값을 반환한다.

sqrt

  • 숫자의 제곱근을 반환한다.

cbrt

  • 주어진 수의 세제곱근을 반환한다.

round

  • 입력값을 반올림한 수와 가장 가까운 정수 값을 반환한다.

ceil

  • 주어진 숫자보다 크거나 같은 숫자 증 가장 작은 숫자를 integer로 반환한다.

floor

  • 주어진 숫자와 같거나 작은 정수 중에서 가장 큰 수를 반환한다.

trunc

  • 주어진 값의 소수부분을 제거하고 숫자의 정수부분을 반환한다.

sign

  • 주어진 수의 부호를 나타내는 +/-1을 반환한다.

log2

log10

exp

  • x를 인수로 하는 e^x 값을 반환한다.

random

  • 0 이상 1 미만의 구간에서 근사적으로 균일한(approximately uniform) 부동소숫점 의사난수를 반환한다.

코딩테스트를 위한 기초 Javascript 문법 정리 String

코딩테스트에서 자주 사용하는 String methods (참고자료)

  • https://developer.mozilla.org/ko/docs/Web/JavaScript/Reference/Global_Objects/String
  • 여러 메소드의 기초 사용법을 유형별로 정리한다.

문자열의 길이

length

  • 문자열의 길이를 나타낸다.

특정 인덱스의 데이터 반환

at

charAt

  • 문자열에서 특정 인덱스에 위치하는 유니코드 단일문자를 반환한다.

charCodeAt

  • 주어진 인덱스에 대한 UTF-16 코드를 나타내는 0 ~ 65535 사이의 정수를 반환한다.

문자열 병합

concat

  • 매개변수로 전달된 모든 문자열을 호출 문자열에 붙인 새로운 문자열을 반환한다.

문자열 탐색 (조건문)

startsWith

  • 어떤 문자열이 특정 문자로 시작하는 지 확인하여 결과를 true 또는 false로 반환한다.
  • 탐색을 시작할 위치를 지정할 수 있다.

endsWith

  • 어떤 문자열에서 특정 문자열로 끝나는지를 확인한다.

includes

  • 하나의 문자열이 다른 문자열에 포함되어 있는지를 판별하고, 결과를 true 또는 false로 반환한다.
  • 대소문자를 구분한다.

indexOf

  • String 객체에서 주어진 값과 일치하는 첫번째 인덱스를 반환한다.

lastIndexOf

  • 일치하는 부분을 fromIndex로부터 역순으로 탐색하여 최초로 마주치는 인덱스를 반환한다.

문자열 탐색 (정규식)

match

  • 문자열이 정규식과 매치되는 부분을 검색한다.
  • 정규식이 아닌 객체를 매개변수로 사용할 수 있다.

matchAll

search

  • 정규 표현식과 String 객체간의 같은 것을 찾는다.
  • 첫번째로 매치되는 것의 인덱스를 반환한다. (없다면 -1)

문자열 채우기

padStart

  • 현재 문자열에 다른 문자열을 채워 주어진 길이를 만족하는 새로운 문자열을 반환한다.
  • 채워넣기는 처음부터 적용된다.

padEnd

  • 현재 문자열에 다른 문자열을 채워 주어진 길이를 만족하는 새로운 문자열을 반환한다.
  • 채워넣기는 끝부터 적용된다.

repeat

  • 문자열을 주어진 횟수만큼 반복해 붙인 새로운 문자열을 반환한다.

trim

  • 문자열 양 끝 공백을 제거한다.
  • 공백이란 모든 공백문자(space, tab, NBSP 등)와 모든 개행문자(LF, CR 등)를 의미한다.

trimStart

  • 문자열 시작 부분의 공백을 제거한다.
  • trimLeft라는 별칭으로 호출 가능하다.

trimEnd

  • 문자열 끝 부분의 공백을 제거한다.
  • trimRight라는 별칭으로 호출 가능하다.

문자열 일부분 변경

replace

  • 어떤 패턴에 일치하는 일부 또는 모든 부분이 교체된 새로운 문자열을 반환한다.
  • 패턴이 문자열인 경우에는 첫번째 문자열만 치환된다.

replaceAll

문자열 자르기

slice

  • 문자열의 일부를 추출하면서 새로운 문자열을 반환한다.

split

  • 문자열을 구분자를 이용해서 여러개의 문자열로 나눈다.

대소문자 변경

toLowerCase

  • 문자열을 소문자로 반환한다.

toUpperCase

  • 문자열을 대문자로 반환한다.

  • test: 대응되는 문자열이 있는지 검사하는 RegExp 메소드, true or false
  • match: 대응되는 문자열을 찾는 string 메소드, 정보를 가지고 있는 배열을 반환하며, 대응되는 문자열이 없을 경우 null을 반환

  • split: 정규식 혹은 문자열로 대상 문자열을 나누어 배열로 반환하는 string 메소드

  • index: 입력된 문자열에서 대응된 부분에 해당하는 인덱스

  • input: 입력된 원본 문자열

  • [0]: 마지막으로 대응된 문자열

  • lastIndex: 다음 검색 시 시작할 인덱스(g 옵션을 설정한 정규식에서만 가능)

  • source: 패턴 문자열

플래그

  • u: 유니코드, 패턴을 유니코드 코드 포인트의 나열로 취급

  • var re = new RegExp(“\w+\s”, “g”);

정규표현식이란?

사용법

  • Hi와 정확히 일치하는 것을 찾는다.
  • Hi 또는 Hello와 정확히 일치하는 것을 찾는다.
  • 소괄호를 통해 묶어주면, match도 되고, group1에 Hi가 지정된다.
  • group1에는 Hi or Hello, group2에는 to와 일치하는 것을 찾으며, 없을 경우 undefined
  • group1은 Hi, Hello, to를 찾으며, group2에는 Hi or Hello, group3에는 to와 일치하는 것을 찾으며, 없을 경우 undefined
  • grey, gray를 찾는다. 이러할 경우 e, a가 그룹으로 묶이는데, 만약 그룹으로 묶고 싶지 않다면 그룹 앞에 ?:을 붙인다.

Quantifiers

  • ?는 없거나 있거나 이다. (gray, gry)
  • 없거나 있거나 많거나 이다. (gry, gray, graay, graaay)

  • a가 2번 나오는 단어를 찾는다. (graay)
  • a가 2~3번 나오는 단어를 찾는다. (graay, graaay)
  • a가 2번 이상 나오는 단어를 찾는다. (graay, graaay, graaaay)

Boundary-type

  • \b를 앞에 사용하면 단어의 맨 앞에서 나오는 Ya만 매칭한다.
  • \b를 뒤에 사용하면 단어의 맨 뒤에서 나오는 Ya만 매칭한다.
  • \b를 앞에 사용하면 단어의 맨 앞에서 나오는 Ya만 빼고 전부 매칭한다.
  • \b를 뒤에 사용하면 단어의 맨 뒤에서 나오는 Ya만 빼고 전부 매칭한다.
  • ^는 문장에서 시작하는 Ya를 찾는다.
  • $는 문장에서 끝나는 Ya를 찾는다.
  • 이때 플래그의 멀티라인을 작성하지 않으면, 문장 전체에서 시작하는 Ya와 문장 전체에서 끝나는 Ya를 찾기 때문에 아무 것도 선택되지 않을 수도 있다.

코딩테스트를 위한 기초 Javascript 문법 정리 Array

코딩테스트에서 자주 사용하는 Array methods (참고자료)

  • https://developer.mozilla.org/ko/docs/Web/JavaScript/Reference/Global_Objects/Array
  • 여러 메소드의 기초 사용법을 유형별로 정리한다.

데이터 반환 (특정 인덱스)

at

  • 해당 인덱스의 요소를 반환한다.
  • .at(-1)을 사용할 수 있다.

다른 메소드와 비교

데이터 추가 및 삭제 (Stack과 Queue)

pop

  • 배열에서 마지막 요소를 제거하고 그 요소를 반환한다.

push

  • 배열의 끝에 하나 이상의 요소를 추가하고, 배열의 길이를 반환한다.

shift

  • 배열에서 첫번째 요소를 제거하고, 제거된 요소를 반환한다.

unshift

  • 새로운 요소를 배열의 맨 앞쪽에 추가한다.

데이터 탐색 (조건문)

some

  • 배열 안의 어떤 요소라도 주어진 판별식을 통과하는 지 테스트한다.

every

  • 배열 안의 모든 요소가 주어진 판별 함수를 통과하는 지 테스트한다.
  • 거짓을 반환하는 요소를 찾을 때까지 배열에 있는 각 요소에 대해 1번씩 실행한다.

includes

  • 배열이 특정 요소를 포함하고 있는지 판별한다.
  • 탐색을 시작할 위치를 지정할 수 있다.

데이터 반환 (조건문)

find

  • 주어진 판별식을 만족하는 첫번째 요소의 값을 반환한다.

findIndex

  • 판별식을 만족하는 첫번째 요소에 대한 인덱스를 반환한다.

indexOf

  • 배열에서 지정된 요소를 찾을 수 있는 첫번째 인덱스를 반환하며, 없다면 -1을 반환한다.
  • 탐색을 시작할 위치를 지정할 수 있다.

lastIndexOf

  • 배열에서 주어진 값을 찾을 수 있는 마지막 인덱스를 반환하며, 없다면 -1을 반환한다.

배열 구조 변경

concat

  • 기존 배열을 변경하지 않으며, 추가된 새로운 배열을 반환한다.

flat

  • 모든 하위 배열 요소를 지정한 깊이까지 재귀적으로 이어붙인 새로운 배열을 생성한다.

flatMap

  • 매핑함수를 사용해 각 엘리먼트에 대해 map을 수행 후, 결과를 새로운 배열로 평탄화 한다.

데이터 변경

fill

  • 배열의 시작부터 끝까지 정적인 값 하나로 채운다.

slice

  • 특정 배열의 처음부터 끝까지 얕은 복사하여 새로운 배열로 반환한다.

splice

  • 배열의 기존 요소를 삭제 또는 교체하거나 새 요소를 추가하여 배열의 내용을 변경한다.

sort

  • 기본 정렬 순서는 문자열의 유니코드 코드 포인트를 따른다.

reverse

  • 배열의 순서를 반전한다.

배열로 변경, 혹은 배열을 변경

from

  • 유사 배열 객체나 반복 가능한 객체를 얕게 복사해 새로운 Array를 만든다.
  • Set에서 배열 만들기

join

  • 배열의 모든 요소를 연결해 하나의 문자열로 만든다.

toString

  • 지정된 배열 및 요소를 나타내는 문자열을 반환한다.

배열 데이터 반복

map

  • 배열 내의 모든 요소 각각에 대해 주어진 함수를 호출한 결과를 모아 새로운 배열로 반환한다.

filter

  • 주어진 함수의 테스트를 통과하는 모든 요소를 모아 새로운 배열로 반환한다.

reduce

  • 각 요소에 대해 주어진 함수를 실행하고, 하나의 결과 값을 반환한다.

Javascript call, apply, 그리고 bind 공부

  • https://www.youtube.com/watch?v=KfuyXQLFNW4&list=PLZKTXPmaJk8JZ2NAC538UzhY_UNqMdZB4&index=13

Call

  • 모든 함수에서 사용할 수 있으며, this를 특정한 값으로 지정한다.
  • 첫 번째 매개 변수는 this로 사용할 값이며, 나머지 뒤의 매개 변수는 함수의 파라매터

Apply

  • 함수의 매개변수를 처리하는 방법을 제외하면 call과 완전히 동일하다.
  • call은 일반적인 함수와 마찬가지로 매개 변수를 직접 받지만, apply는 매개변수를 배열로 받는다.

Bind

  • 함수의 this 값을 영구히 바꾼다.
