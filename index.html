Javascript 정규표현식 정리

정규표현식이란?

  • pattern과 flag를 이용한다.

사용법

  • Hi와 정확히 일치하는 것을 찾는다.
  • Hi 또는 Hello와 정확히 일치하는 것을 찾는다.
  • 소괄호를 통해 묶어주면, match도 되고, group1에 Hi가 지정된다.
  • group1에는 Hi or Hello, group2에는 to와 일치하는 것을 찾으며, 없을 경우 undefined
  • group1은 Hi, Hello, to를 찾으며, group2에는 Hi or Hello, group3에는 to와 일치하는 것을 찾으며, 없을 경우 undefined
  • grey, gray를 찾는다. 이러할 경우 e, a가 그룹으로 묶이는데, 만약 그룹으로 묶고 싶지 않다면 그룹 앞에 ?:을 붙인다.
  • /gr(e|a)y/와 동일하며, 찾고싶은 문자열 집합체를 작성한다.
  • 하나라도 만족하는 것을 찾는다.
  • /gr[a-f]y/와 동일하다
  • a부터 f까지를 제외한 문자열을 찾는다.

Quantifiers

  • ?는 없거나 있거나 이다. (gray, gry)
  • 없거나 있거나 많거나 이다. (gry, gray, graay, graaay)
  • 하나 있거나 많거나 이다. (gray, graay, graaay)
  • a가 2번 나오는 단어를 찾는다. (graay)
  • a가 2~3번 나오는 단어를 찾는다. (graay, graaay)
  • a가 2번 이상 나오는 단어를 찾는다. (graay, graaay, graaaay)

Boundary-type

  • \b를 앞에 사용하면 단어의 맨 앞에서 나오는 Ya만 매칭한다.
  • \b를 뒤에 사용하면 단어의 맨 뒤에서 나오는 Ya만 매칭한다.
  • \b를 앞에 사용하면 단어의 맨 앞에서 나오는 Ya만 빼고 전부 매칭한다.
  • \b를 뒤에 사용하면 단어의 맨 뒤에서 나오는 Ya만 빼고 전부 매칭한다.
  • ^는 문장에서 시작하는 Ya를 찾는다.
  • $는 문장에서 끝나는 Ya를 찾는다.
  • 이때 플래그의 멀티라인을 작성하지 않으면, 문장 전체에서 시작하는 Ya와 문장 전체에서 끝나는 Ya를 찾기 때문에 아무 것도 선택되지 않을 수도 있다.

Character classes

  • 줄바꿈 문자를 제외한 모든 문자열을 선택한다.
  • 만약에 .를 진짜로 찾고 싶은것이라면, \를 이용해야 한다.
  • 특수 문자인 경우에는 \ 다음에 작성을 해야된다.
  • 숫자를 전부 찾는다.
  • 숫자를 제외하고 전부 찾는다.
  • 문자를 전부 찾는다.
  • 문자를 제외하고 전부 찾는다.
  • 공백을 전부 찾는다.

  • -it는 컨테이너를 연 다음 cli를 사용하겠다는 의미이다
  • 그 후 node 명령어가 실행되므로 입력 화면이 나오는 것이다

  • docker exec -it (docker ps 명령어로 확인한 컨테이너 명) bash

docker-compose

docker-compose 내용 설명

  • ports: 는 외부에 개방할 포트이다

  • run 실행마다 작성해야되었던 명령어를 문서로 미리 작성한 것이다

  • backend에서는 volume을 사용하기 때문에 volumes:를 작성한다

  • 환경 변수도 compose에서 environment:로 미리 설정할 수 있다

docker-compose 실행

  • docker-compose up을 실행
  • docker-compose up -d를 하면 뒤에서 알아서 컨테이너 생성 및 실행

Javascript call, apply, 그리고 bind 공부

  • https://www.youtube.com/watch?v=KfuyXQLFNW4&list=PLZKTXPmaJk8JZ2NAC538UzhY_UNqMdZB4&index=13

Call

  • 모든 함수에서 사용할 수 있으며, this를 특정한 값으로 지정한다.
  • 첫 번째 매개 변수는 this로 사용할 값이며, 나머지 뒤의 매개 변수는 함수의 파라매터

Apply

  • 함수의 매개변수를 처리하는 방법을 제외하면 call과 완전히 동일하다.
  • call은 일반적인 함수와 마찬가지로 매개 변수를 직접 받지만, apply는 매개변수를 배열로 받는다.

Bind

  • 함수의 this 값을 영구히 바꾼다.
